# Promptory 2026 Quick Audit 검증 설계서

**버전**: 2026.03.31  
**검증 목표**: Quick Audit Pack (49만원) 시장 적합성 및 가격 최적화  
**실험 기간**: 2주 (Week 1-2) + 분석 1주 (Week 3)

---

## 1. 검증 가설 (Hypotheses to Validate)

### H1: 시장-제품 적합성 (Problem-Solution Fit)
> **한국 B2B 중소기업(20-50인) 마케팅 리드는 URL 기반 사이트 진단 초안에 29-99만원을 지불할 의사가 있다**

- 세부 가설 H1-A: 초안 병목을 겪는 팀은 유료 진단의 가치를 인정한다
- 세부 가설 H1-B: Slack 사용 여부는 기술 수용도와 상관관계가 있다

### H2: 가격 민감도 (Price Elasticity)
> **49만원은 수용 가능한 진입 가격이며, 79만원에서도 전환율이 50% 이상 유지된다**

| 가격대 | 가설 전환율 | 타겟 수익성 |
|--------|-------------|-------------|
| 29만원 | 25-30% | 고객 확보 우선 |
| 49만원 | 15-20% | **채택 목표** |
| 79만원 | 8-12% | 프리미엄 포지셔닝 |
| 99만원 | 3-5% | 고수익 검증 |

### H3: 업셀 경로 (Upsell Path)
> **Quick Audit 구매 고객의 30% 이상이 Core Package(300만원+)로 전환된다**

### H4: 채널 효율성 (Channel Efficiency)
> **LinkedIn 직접 아웃리치 대비 기존 네트워크 추천의 전환율이 2배 이상 높다**

## 2. 가격 A/B/C 실험 설계

### 2.1 실험 구조 (Multi-Arm Bandit)

```
기간: 2주 (10영업일)
총 타겟: 60개 업체 (20개 × 3가지 가격대)

[그룹 A] 29만원 그룹: 20개 업체
[그룹 B] 49만원 그룹: 20개 업체  
[그룹 C] 79만원 그룹: 20개 업체
```

### 2.2 무작위화 (Randomization)

| 변수 | 제어 방식 |
|------|----------|
| 업체 규모 | 20-50인으로 고정 (외생 변수 제어) |
| 산업군 | B2B SaaS/제조/컨설팅 균등 배분 |
| 접촉 채널 | LinkedIn DM (동일) |
| 메시지 | 가격 외 동일 템플릿 사용 |
| 시점 | 동일 주차 내 분산 접촉 |

### 2.3 실험 프로세스

```
Day 1-2: 60개 업체 리스트업 (LinkedIn Sales Navigator)
Day 3-5: 초기 아웃리치 (가격별 무작위 할당)
Day 6-10: 미팅/데모 진행
Day 11-14: 제안서 발송 및 협상
Day 15: 데이터 수집 마감
Day 16-21: 분석 및 보고
```

### 2.4 통계적 유의성 (Statistical Power)

```
목표: 95% 신뢰수준, 10% 허용 오차
최소 샘플: 그룹당 20개 = 총 60개

t-test 조건:
- 그룹 A (29만) vs 그룹 B (49만): p < 0.05 확인
- 그룹 B (49만) vs 그룹 C (79만): p < 0.05 확인
```

## 3. 지표 체계 (Metrics Framework)

### 3.1 핵심 지표 (North Star Metrics)

| 지표 | 정의 | 목표값 | 측정 방식 |
|------|------|--------|----------|
| **Paid Audit Rate** | Quick Audit 유료 구매율 | 15-20% | 구매 / 총 미팅 |
| **Core Upsell Rate** | Core Package 전환율 | 30%+ | Core 계약 / Audit 구매 |
| **CAC Recovery** | CAC 회수 기간 | <30일 | Quick Audit 수익으로 영업비용 회수 |

### 3.2 단계별 지표 (Funnel Metrics)

```
[인지] 클릭률 (CTR)
  ↓ 5-10%
[관심] 상담 요청률 (Lead Conversion)
  ↓ 30-50%
[평가] 데모 미팅 완료률
  ↓ 60-80%
[의도] 제안서 수락률 (Offer Acceptance)
  ↓ 40-60%
[구매] 결제 완료율 (Purchase Rate) ← 핵심
  ↓ 30%
[확장] Core Package 전환율 (Core Upsell)
```

### 3.3 측정 세부 정의

| 단계 | 이벤트 | 데이터 소스 | 주요 질문 |
|------|--------|-------------|-----------|
| 인지 | 랜딩 페이지 방문 | GA4/Amplitude | "어떤 채널이 효율적인가?" |
| 관심 | /contact 폼 제출 | 내부 DB | "어떤 pain point가 강한가?" |
| 평가 | 캘린더 예약 완료 | Calendly | "얼마나 빠르게 결정하는가?" |
| 의도 | 제안서 발송 후 반응 | CRM | "가격 거절 이유는 무엇인가?" |
| 구매 | Toss Payments 완료 | Toss + Slack 알림 | "실제 결제까지 걸리는 시간?" |
| 확장 | Core Package 상담 시작 | CRM 수동 기록 | "Audit 품질 vs 전환 상관관계?" |

### 3.4 정성적 피드백 수집

**인터뷰 템플릿 (Post-Purchase)**
```
1. 어떤 pain point가 가장 급했나요? (초안/비교/보고/기타)
2. 49만원 결정에 걸린 시간은? (즉시/하루/일주일)
3. 만약 79만원이었다면 구매했을까요? (예/아니오/모르겠음)
4. Core Package(300만원)로 업그레이드 의향은? (1-10점)
5. 주변에 추천할 의향은? (NPS: 0-10점)
```

## 7. 확장 채널 로드맵

### 7.1 Phase 1: Slack-First (현재)
**기간**: 2026 Q1-Q2  
**타겟**: Slack 사용 중인 중소기업  
**검증**: Quick Audit 실험으로 PMF 확인

### 7.2 Phase 2: Microsoft Teams (조건부)
**기간**: 2026 Q3 (Quick Audit 50건+ 달성 시)  
**전제조건**:
- Slack 고객 중 20%+가 "Teams도 쓴다" 언급
- Teams App 개발 비용 < 200만원
- Teams용 Quick Audit 10건 사전 예약 확보

**개발 범위**:
- Teams Bot Framework 연동
- Slack과 동일한 명령어 구조
- App Home → Teams Tab 변환

### 7.3 Phase 3: Web-First (독립형)
**기간**: 2026 Q4 또는 2027  
**전제조건**:
- Core Package 20개사 이상 운영
- Slack 미사용 고객의 churn이 30%+
- Web 임베드 수요 5건+/월

**개발 범위**:
- Embeddable Widget (고객 사이트용)
- Standalone Web Dashboard
- API-First 아키텍처 전환

### 7.4 채널 우선순위 결정 트리

```
[현재: Slack-First]
    ↓
Q: Teams 수요가 20%+인가?

    ↓
```
