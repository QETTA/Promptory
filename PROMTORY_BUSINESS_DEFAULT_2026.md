# 프롬프토리 2026 사업 디폴트 (One-Page Business Plan)

## One-Line Pitch
**"홈페이지는 있는데 초안이 밀리는 중소기업 팀에게, URL 기반 진단과 실행 초안을 Slack 안에서 바로 이어주는 맞춤형 Agent 패키지"**

---

## Problem (Pain)
한국 B2B 중소기업(20~200인, 마케팅 1~5명)의 반복 실무 병목:
1. **초안 병목**: 비교표, 카피, 브리프, 보고 정리가 계속 밀림
2. **맥락 단절**: 범용 AI 툴은 회사 URL/채널/목표 기준으로 바로 이어지지 않음
3. **협업 단절**: 결과가 웹툴에 남고 Slack 팀 대화 안에서 공유·수정이 안 됨
4. **반복 비용**: 대표/팀장이 매주 사이트 보고→경쟁사 훑고→방향 정리→보고 작성을 손으로 반복

---

## Solution (Product)
**Promptory = 맞춤형 Slack Agent Package**

고객이 느끼는 가치: "AI 툴을 도입했다"가 아니라 **"우리 팀 Slack에 초안 만드는 사람 하나 붙였다"**

### 핵심 패키지
| 패키지 | 대상 | 핵심 출력물 |
|--------|------|------------|
| Website Diagnosis Agent | 중소기업 마케팅팀 | 사이트 병목 분석, 경쟁사 비교표, CTA 초안, 보고용 요약 |
| Campaign Brief Agent | 기획/운영팀 | 회의→브리프 변환, 메시지 방향, 액션 리스트 |
| Korea Local Ops Agent | 외국계 한국지사 | KR/EN executive summary, HQ 공유용 action memo |

### 유입 상품 (Quick Audit Pack)
- **가격**: 1회 49~99만원
- **구성**: URL 기반 1차 진단 + 비교표 초안 + 방향 제안
- **목적**: 진입, 신뢰 확보, 핵심 패키지 업셀

---

## Business Model (수익모델)
**구축형 + 운영형 + 업셀**

### 핵심 상품: Core Slack Agent Package
- **구축비**: 300~500만원 (초기 세팅, Slack 연동, 워크플로 고정)
- **월 운영비**: 79~99만원
- **포함**: Slack DM/채널/모달/App Home 기본 세팅, 1개 핵심 워크플로, 저장/이어보기

### 고수익 상품: Growth/Multi-flow Package
- **구축비**: 700만~1,200만원
- **월 운영비**: 149~249만원
- **대상**: 외국계 한국지사, 복수 팀, KR/EN summary 필요 기업

### 후속 업셀
- 추가 패키지 (+월 40~60만원)
- 결과물 포맷 커스터마이징
- 정기 운영 리포트
- 팀별 App Home 분리

---

## Go-to-Market (영업 전략)
**직접 영업 + 맞춤형 패키지 판매**

### 영업 퍼널
```
[진입] Quick Audit Pack (49~99만원) → 
[핵심] Core Package (구축 300~500만 + 월 79~99만) → 
[운영] 월 구독 유지 → 
[업셀] 추가 패키지/포맷/리포트
```

### 핵심 메시지
> "Slack에 URL을내면, 진단부터 실행 초안까지 팀 대화 안에서 끝납니다"

---

## Market & ICP (시장 & 타겟)

### 1차 핵심 ICP
**한국 B2B 중소기업의 마케팅/사업기획 리드**
- 인원: 20~200명
- 마케팅/기획 인력: 1~5명
- 특징: 대표/팀장이 초안 품질을 직접 챙김, 비교표/카피/보고 초안이 계속 밀림
- 도구: Slack을 내부 커뮤니케이션 허브로 사용

### 2차 확장 ICP
**외국계 한국지사의 마케팅/채널/BD 리드**
- KR/EN 동시 요약 필요
- HQ 보고용 포맷 커스텀 필요

---

## Competitive Advantage (차별성)
1. **Slack-native**: 입력부터 출력까지 팀 대화 안에서 완결
2. **URL-first**: 홈페이지/랜딩/문서 링크 하나로 진단 시작
3. **맞춤형**: 팀의 입력물, 질문 흐름, 출력 형식을 세팅 단계에서 고정
4. **구축형 접근**: 범용 SaaS가 아닌 "우리 팀용으로 셋업해주는 패키지"로 ARPA 극대화
